代码审核常见问题，你中了几个呢？

发表于2025-12-09|更新于2025-12-09|通用编程
|总字数:1.8k|阅读时长:8分钟|浏览量:

最近一年，做了很多代码审核工作，团队中的成员有刚毕业的，也有久战沙场的。但不管是新人，还是老手，都会发现一些问题。有些英语不好，类名、变量名、方法名乱取，让我苦不堪言；有些不注重代码性能，for 循环从不省着用；也有些喜欢 copy/paste, 同样的代码满天飞。还有很多让我头疼的，就不一一例举了。

接下来，我就以Qt C++ 实现登陆模块的代码为例，来例举些常见问题:
注：非公司的实际项目代码，只是用来例举下常见问题

英文命名太糟糕: 难理解

中式英语字面直译：别扭绕口

名字上下文不一致性：混乱

类中存在多个重复方法：难维护

if嵌套层数太深：难阅读

非专有名词的变量乱缩写：理解烧脑

创建无效变量：浪费

不知提前结束循环：性能损耗

函数代码行数太多：看到下面，忘记上面
