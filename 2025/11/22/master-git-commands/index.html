Git 简介

Git 相信大家并不陌生，它是目前全球最流行的分布式版本控制系统，由 Linux 内核创始人林纳斯・托瓦兹（Linus Torvalds）于 2005 年开发，最初用于管理 Linux 内核的代码开发，如今已成为软件开发、团队协作中不可或缺的工具。它的核心作用有以下几个，但并不限于这几个：

  • 追踪文件的修改历史
  • 协调多人协作开发
  • 回滚错误版本
  • 管理代码分支

Git 并不是开发的专属，你也可以用它来管理你的写作，用它来追踪你的作品各个版本或历史修改记录; 可以说，很多领域，它都可以帮上你的忙。

Git 可视化工具，我主要使用命令行工具 + Sourcetree(支持 macOS , windows)。很多 Windows 用户喜欢用 TortoiseGit.

Git配置

在开始Git之旅之前，我们需要设置一下Git的配置变量，这是一次性的工作。这些设置会在用户文件(用户主目录的.gitconfig)或系统文件(eg. /etc/gitconfig) 中做永久的记录。

配置用户信息

Git 的配置采用分层级的设计，从大到小分为系统级（system）、全局级（global）、本地级（local），层级越细优先级越高（本地 > 全局 > 系统）。git config -l 则是查看所有生效的配置（合并三层配置后的结果）。

  • –system 会将配置写入系统级 Git 配置文件，作用域是整个操作系统的所有用户和所有 Git 仓库
  • –global 会将配置写入当前用户的全局配置文件，作用域是当前登录用户的所有 Git 仓库。这是最常用的配置层级
  • –local 本地配置文件（仓库下的 .git/config）属于当前用户的项目文件

以上过程是不是步骤很多，我们也可以使用让其简单一点，可以达到同样的目的：

如果不是 fast forward 时，git merge 会自动额外的一个 merge commit。尤其在有冲突的情况下，会把它人的 commits 合成一个 merge commit 。所以在非 fast forward 合并时，强烈推荐使用 git rebase 。
